READLINE :
prototype : char *readline(const char *prompt);
Utilisation : La fonction readline est utilisée pour lire une ligne de texte entrée par 
			  l'utilisateur depuis l'entrée standard (stdin). Elle affiche une invite (prompt), attend que
			  l'utilisateur saisisse une ligne de texte, puis retourne cette ligne sous
			  forme de chaîne de caractères dynamique.
Header : #include <readline/readline.h>
		 #include <readline/history.h>  // (optionnel, si on veut utiliser l'historique des commandes)
Retour : Retourne un pointeur vers une chaîne de caractères contenant l'entrée de l'utilisateur.
		 La chaîne est allouée dynamiquement (avec malloc), donc il faut la libérer avec free après utilisation.
		 Retourne NULL en cas d'erreur ou si l'entrée est EOF.
Exemple : 
	int main() {
    char *input;

    // Affiche une invite et attend l'entrée de l'utilisateur
    input = readline("Entrez une commande : ");
    if (input) {
        printf("Vous avez entré : %s\n", input);
        free(input); // Libérer la mémoire allouée par readline
    } else {
        printf("Erreur ou fin de l'entrée.\n");
    }
    return 0;
}

Fork : 
Usage : Crée un nouveau processus en dupliquant le processus courant.
		En gros, cela crée un nouveau processus (le fils), qui s'execute apres l'appel de fork simultanement.
		Le retour differe dans le processus parent (PID du fils) et dans le fils(0).
Prototype : pid_t fork(void);
Exemple : pid_t pid = fork();
Dans le processus parent fork retrourne le PID(identifiant de processus) du processus fils.
Dans le processus fils : fork return 0.
Si une erreur se produit fork renvoi -1.
lors d un appel a fork, le processus fils recoit une copie de l'espace memoire du proccessus parent pour pouvoir travailler.
Cependant, la reelle copie a uniquement lieu quand le fils cherche a faire une modification dynamique.



Wait : 
Usage : Permet au processus parent d'attendre la terminaison d'un processus fils.
Prototype : pid_t wait(int *status);
Exemple : wait(&status);
Si le processus fils est terminer avec succes, return PID du fils.
Sinon return -1 que errno comprend.
status est un pointeur vers un entier dans lequel wait va stocker la code de terminaison du proccessus fils.



Waitpid : Usage : Permet au processus parent d'attendre un processus fils particulier, avec des options pour controler l'attente.
		  Prototype : pid_t waitpid(pid_t pid, int *status, int options);
		  Exemple : waitpid(pid, &status, 0);
		  pid > 0 : Attend la fin du processus dont le PID est pid.
		  pid == 0 : Attend tout les proccessus fils du meme groupe de proccessus.
		  pid == -1 : Attend n importe quel processus fils (equivalent a wait).
		  pid < 1: Attend tout les proccessus fils dont le groupe de processus = |pid|.
		  status = un pointeur vers un entier ou sera stocker le code de terminaison du proccessus.
		  options : Specifie les options pour la fonction waitpid().
		  			0 = Attendre la fin du proccessus fils.
					WNOHANG : Si aucun proccessus fils n a terminer, waitpid retourn immediatement au lieu de bloquer.
					WUNTRACED : Si un processus fils est arrete (pas terminer) la fonction retourne aussi son etat.
		  Si le processus fils est terminer avec succes, return PID du fils.
	      Sinon return -1 que errno comprend.




wait3 :

Wait4 : 
prototype : pid_t wait4(pid_t pid, int *wstatus, int options, struct rusage *rusage);
Utilisation : La fonction wait4 est utilisée pour attendre la terminaison d'un processus
			  enfant spécifique et récupérer des informations détaillées sur son statut de fin et
			  son utilisation des ressources (comme la CPU et la mémoire). Elle est similaire à wait3,
			  mais avec plus de contrôle sur le processus à attendre grâce au paramètre pid, permettant de
			  spécifier un processus particulier ou d'attendre tous les enfants.
Header : #include <sys/types.h>
		 #include <sys/wait.h>
		 #include <sys/resource.h>
Retour : Retourne le PID du processus enfant terminer : Si pid est 0, wait4 attend tous les processus enfants.
		 Si pid est -1, il attend n'importe quel enfant. Si pid est un numéro de processus spécifique, il attend
		 ce processus particulier.
		 retourn -1 en cas d'erreur avec errno.
Paramètres :
pid : Le PID du processus à attendre. Utilisez -1 pour tous les enfants, ou 0 pour les processus enfants directs du parent.
wstatus : Pointeur vers un entier où sera stocké le statut de fin du processus enfant (utilisable avec WIFEXITED, WIFSIGNALED, etc.).
options : Options pour modifier le comportement (par exemple, WNOHANG pour ne pas bloquer si aucun enfant n'a terminé).
rusage : Pointeur vers une structure struct rusage pour récupérer l'utilisation des ressources du processus. Peut être NULL si ces informations ne sont pas nécessaires.



SIGNAL :
Prototype : void (*signal(int sig, void (*handler)(int)))(int);
Utilisation : La fonction signal est utilisée pour définir un gestionnaire de signaux dans un programme C.
			  Elle permet de spécifier une fonction à exécuter lorsqu'un signal particulier est reçu par le processus.
			  Par exemple, elle peut être utilisée pour gérer des signaux comme SIGINT
			  (interruption, souvent envoyé par Ctrl+C) ou SIGTERM (demande de terminaison).
Header : #include <signal.h>
Retour : Retourne l'adresse du gestionnaire de signaux précédent, ou SIG_ERR en cas d'erreur.
		 Si le gestionnaire est correctement défini, la fonction retourne un pointeur vers le précédent gestionnaire.
		 Si une erreur se produit, elle retourne SIG_ERR et errno est défini.
Parametre : sig : Le signal à capturer. Il s'agit généralement d'un nombre entier correspondant à un signal standard comme SIGINT, SIGTERM, SIGKILL, etc.
			handler : Un pointeur vers la fonction qui doit être appelée lorsqu'un signal spécifique est reçu. Cette fonction doit
		    avoir un paramètre d'entier (int), représentant le signal reçu. Si handler est SIG_IGN, le signal sera ignoré,
		    et si handler est SIG_DFL, le comportement par défaut du signal sera utilisé.

Execve : Usage : Remplace le processus courant par un autre programme. En gros, execute un programme en remplacant le
					processus courant. C'est souvent utiliser pour lancer un progranne avec des variables d'environnement.
		 Prototype : int execve(const char *pathname, char *const argv[], char *const envp[]);
		 Exemple : execve("/bin/ls", args, env);
		 La fonction execve est une des plus importantes du systeme Unix/Linux
		 elle remplace un programme en cours d'execution par un autre programme.
		 pathname : correspond au chemin absolu ou relatif du programme a executer.
		 argv : un tableau de chaine de caractere qui represente les arguments du programme a executer
		 le premier element est souvent le nom du programme a executer et le dernier est NULL.
		 Exemple : pour executer ls -l, argv serait ["ls", "-l", NULL]
		 envp = un tableau de chaine de caractere representant l environnement du programme a executer, chaques 
		 chaines doivent etre sous la forme clé=valeur : ["PATH=/usr/bin", "HOME=/home/user", NULL]
		 si elle echoue la fonction renvoie -1 qui sera transmissible via errno.
		 Les descripteurs de fichier ouverts dans le programme parent seront transmis dans le programme fils, a moins
		 d'etre fermer avant.
		 les variables d'environnement seront transise via le tableau envp.
		 On combine souvent cette fonction avec fork, fork va cree un processus fils, et execve va permettre d'associer ce
		 processus fils a un executable, exemple, on veux executer ls dans un programme : 
		 on fait un pid_t	pid = fork(); //cree un processus fils.
		 					if (pid == 0)
		 						execve("/bin/ls", argv, envp)   //associe au processus fils l'executable ls.
							else
								wait(NULL);  //attend que le processus fils soit terminer.

Pipe : Usage : Crée un pipe(un canal de communication unidirectionnel entre deux processus).
				Crée un canal entre deux processus pour permettre la communication.
	   Prototype : int pipe(int pipefd[2]);
	   Exemple : int pipefd[2];
	   			 pipe(pipefd);
	   Pipe est essentiel car il permet de cree une communication entre deux processus par un mecanisme de tuyau(pipe).
	   pipefd[2] est un tableau de deux descripteur de fichiers.
	   pipefd[0] = descripteur pour lire a partir du pipe.
	   pipefd[1] = descripteur pour ecrire dans le pipe.
	   Si pipe reussit, il return 0 et rempli le tableau pipefd[] avec les deux descripteur de fichiers.
	   En car d erreur pipe return -1, avec son erreur errno associer.
	   pipe est unidirectionnel donc les donnees peuvent circuler uniquement d un sens a la fois.
	   Pipe fonctionne entre deux processus, c'est pour ca que souvent fork est utiliser avec pour pouvoir cree deux processus simultane.
	   Exemple concret pour le projet :
	   si tu veux executer des commandes avec un pipe dans un shell, comme par exemple ls | grep txt, cela 
	   implique la creation de deuc proccessus avec un pipe entre eux :
	   1. Le processus fils 1 executera ls et ecrira dans le pipe.
	   2. le proccessus fils 2 executera grep txt et lira les donnees du pipe.
	   Ce type de mecanisme est realiser en creant un pipe entre les deux processus et en redirigeant les flux entree/sortie.

    unsigned short d_reclen;    // Longueur de l'enregistrement
    unsigned char  d_type;      // Type de fichier (si supporté)
    char           d_name[256]; // Nom du fichier
};



CLOSEDIR : 
Prototype : int closedir(DIR *dirp);
Utilisation : La fonction closedir est utilisée pour fermer un répertoire précédemment ouvert avec opendir().
			  Cela libère les ressources associées à l'accès au répertoire. Elle est essentielle pour éviter 
			  les fuites de mémoire et les problèmes de gestion de fichiers.
Header : #include <dirent.h>
Parametre : dirp : Un pointeur vers la structure DIR obtenue avec opendir().
Retour : Retourne -1 en cas d'erreur.
		 Retourne 0 en cas de succes.



strerror : Usage : Renvoi une chaine de caractere d'écrivant l'erreur associé a un code erreur errno.
		   Prototype : char *strerror(int errnum);
		   Exemple : printf("Erreur : %s\n", strerror(errno));




Perror : Usage : Affiche un message d'erreur basé sur la variable errno, en gros affiche un message d'erreur
					avec une description textuelle en cas d'échec.
		 Prototype : void perror(const char *s);
		 Exemple : perror("Erreur lors de l'ouverture du fichier");
		 la variable errno est une variable globale qui contient le dernier code erreur défini par une fonction systeme.
		 donc dans notre cas, perror("Erreur lors de l'ouverture du fichier");
		 					  cela va afficher : Erreur lors de l'ouverture du fichier: No such file or directory




ISATTY :
Prototype : int isatty(int fd);
Utilisation : La fonction isatty permet de vérifier si un descripteur de fichier (fd) correspond à un
			  terminal (TTY). Elle est utile pour détecter si un programme interagit avec un utilisateur
			  en ligne de commande ou s'il est redirigé vers un fichier ou un pipe.
Header : #include <unistd.h>
Parametre : fd : Un descripteur de fichier (exemples courants : STDIN_FILENO, STDOUT_FILENO, STDERR_FILENO).
Retour : Retourne 1 si fd est associé à un terminal.
		 Retourne 0 sinon (ex. redirection vers un fichier ou un pipe).



TTYNAME :
Prototype : char *ttyname(int fd);
Utilisation : La fonction ttyname est utilisée pour obtenir le nom du périphérique terminal
			  associé à un descripteur de fichier donné. Elle est souvent utilisée pour identifier
			  le terminal avec lequel un programme interagit, par exemple, pour obtenir le nom d'un
			  terminal interactif sur lequel un utilisateur est connecté. Cela peut être utile dans des
			  applications où le programme doit se comporter différemment selon qu'il est connecté à un
			  terminal physique ou redirigé vers un fichier.
Header : #include <unistd.h>
		 #include <stdio.h>
Parametre : fd : Un descripteur de fichier, généralement STDIN_FILENO, STDOUT_FILENO, ou STDERR_FILENO,
			mais il peut également être un descripteur de fichier ouvert manuellement pour un terminal
			particulier.
Retour : Retourne une chaîne de caractères contenant le chemin du terminal associé au descripteur de fichier fd.
		 Retourne NULL en cas d'erreur, et l'erreur est définie par errno.




TTYSLOT :
Prototype : int ttyslot(void);
Utilisation : La fonction ttyslot renvoie un entier représentant le numéro de la ligne de terminal (tty)
			  utilisé pour l'exécution du programme. Cela peut être utile pour déterminer l'environnement
			  du terminal actuel, surtout dans un contexte multi-utilisateur, pour savoir quel terminal est
			  associé à un processus particulier.
Header : #include <unistd.h>
Parametre : void
Retour : Retourne un entier, le numéro de la ligne de terminal (tty) utilisé par le processus.
		 Retourne -1 si le terminal actuel n'est pas valide ou s'il y a une erreur.




IOCTL :
Prototype : int ioctl(int fd, unsigned long request, ...);
Utilisation : La fonction ioctl est utilisée pour contrôler les périphériques via un descripteur de fichier.
			  Elle permet d'envoyer des commandes spécifiques à des périphériques (par exemple des périphériques de terminal,
			  de disque, de réseau) pour configurer leur comportement ou obtenir des informations spécifiques.
			  Elle est souvent utilisée pour des opérations qui ne peuvent pas être réalisées par des appels système standard
			  comme read() ou write().
Header : #include <sys/ioctl.h>
Parametre : fd : Un descripteur de fichier (par exemple, un fichier, un périphérique, ou un terminal).
			request : Une valeur (souvent définie comme une constante) indiquant la commande à exécuter.
		    Les paramètres supplémentaires (troisième argument) dépendent de la commande request.
			Par exemple, un tampon de données ou une structure de configuration.
Retour : Retourne 0 en cas de succès.
		 Retourne -1 en cas d'erreur et définit errno.




GETENV : 
Prototype : char *getenv(const char *name);
Utilisation : La fonction getenv permet d'obtenir la valeur d'une variable d'environnement à partir du tableau
			  d'environnement du processus. Elle est couramment utilisée pour récupérer des informations de configuration,
			  telles que les variables de chemin (PATH), de configuration du système, ou d'authentification.
Header : #include <stdlib.h>
Parametre : name : Le nom de la variable d'environnement à récupérer,
			sous forme de chaîne de caractères (par exemple "HOME", "PATH", etc.).
Retour : Retourne un pointeur vers une chaîne de caractères contenant la valeur de la variable d'environnement spécifiée.
		 Retourne NULL si la variable d'environnement n'est pas définie.




TCSETATTR :
Prototype : int tcsetattr(int fd, int action, const struct termios *termios_p);
Utilisation : La fonction tcsetattr est utilisée pour modifier les attributs du terminal associé au descripteur de fichier fd.
			  Cela permet de configurer des aspects du terminal comme le mode de saisie (canonique ou non-canonique),
			  l'écho des caractères, les délais de lecture, et d'autres paramètres de contrôle.
			  Elle est couramment utilisée pour manipuler l'entrée/sortie de terminal dans des applications interactivas
			  (par exemple, pour capturer les touches d'un utilisateur de manière instantanée ou pour configurer l'environnement
			  du terminal).
Header : #include <termios.h>
Parametre : fd : Le descripteur de fichier associé au terminal dont les attributs doivent être modifiés. Cela pourrait être STDIN_FILENO (entrée standard), STDOUT_FILENO (sortie standard), ou un autre descripteur de fichier pointant vers un périphérique terminal.
		    action : Spécifie l'action à prendre. Elle peut être l'une des valeurs suivantes :
			TCSANOW : Les modifications sont appliquées immédiatement.
			TCSADRAIN : Les modifications sont appliquées une fois que toutes les sorties en cours ont été envoyées.
			TCSAFLUSH : Les modifications sont appliquées une fois que toutes les sorties en cours ont été envoyées et que toutes les entrées non lues ont été supprimées.
			termios_p : Un pointeur vers une structure termios qui contient les nouveaux paramètres du terminal à appliquer.
Retour : Retourne 0 en cas de succès.
		 Retourne -1 en cas d'erreur et définit errno.



TCGETATTR :
Prototype : int tcgetattr(int fd, struct termios *termios_p);
Utilisation : La fonction tcgetattr est utilisée pour récupérer les attributs actuels du terminal associé au descripteur de
			  fichier fd. Elle permet de lire la configuration du terminal (comme les paramètres d'entrée et de sortie,
			  les modes de contrôle, les délais, etc.). Cette fonction est souvent utilisée pour sauvegarder la configuration
			  d'un terminal avant de la modifier (par exemple, avant de passer en mode non-canonique ou de désactiver l'écho
			  des caractères), afin de pouvoir la restaurer plus tard avec tcsetattr.
Header : #include <termios.h>
Parametre : fd : Le descripteur de fichier associé au terminal dont les attributs doivent être récupérés. Cela peut être STDIN_FILENO, STDOUT_FILENO, ou STDERR_FILENO (ou tout autre descripteur de fichier pour un terminal spécifique).
		    termios_p : Un pointeur vers une structure termios où les attributs du terminal seront copiés.
Retour : Retourne 0 en cas de succès.
		 Retourne -1 en cas d'erreur et définit errno.



TGETENT :
Prototype : int tgetent(char *bp, const char *name);
Utilisation : La fonction tgetent est utilisée pour charger une entrée de la base de données des terminaux (termcap)
			  dans la mémoire afin de pouvoir l'utiliser pour obtenir des informations sur un terminal spécifique,
			  comme ses capacités (par exemple, la gestion des couleurs, des modes d'affichage, etc.).
			  Elle est utilisée dans des programmes qui nécessitent des informations détaillées sur le terminal pour
			  ajuster leur comportement (par exemple, des applications en ligne de commande avec des
			  interfaces utilisateur enrichies).
Header : #include <termcap.h>
Parametre : bp : Un pointeur vers un tampon où l'entrée du terminal sera chargée. Ce tampon est utilisé par la fonction
			pour stocker l'entrée du terminal lue. Il doit être suffisamment grand pour contenir les données.
			name : Le nom du terminal (par exemple, "xterm", "vt100", etc.). Ce nom doit correspondre à une entrée
			dans la base de données termcap ou dans le fichier de base de données des terminaux.
Retour : Retourne 1 si l'entrée du terminal a été trouvée et chargée avec succès.
		 Retourne 0 si l'entrée du terminal spécifiée n'existe pas.
		 Retourne -1 en cas d'erreur.




TGETFLAG :
Prototype : int tgetflag(const char *id);
Utilisation : La fonction tgetflag est utilisée pour vérifier si un terminal possède une capacité spécifique qui est
			  représentée sous forme de drapeau (flag) dans la base de données termcap. Contrairement à tgetnum
			  (qui récupère des valeurs numériques) et tgetstr (qui récupère des chaînes de caractères),
			  tgetflag retourne simplement un booléen (1 si la capacité est présente, 0 sinon).
Header : #include <termcap.h>
Parametre : id : Une chaîne de 2 caractères représentant le code de la capacité à tester (par exemple,
			"bs" pour backspace disponible, "am" pour auto-wrap, etc.).
Retour : Retourne 1 si le terminal supporte la capacité spécifiée.
		 Retourne 0 si la capacité n'est pas présente.	




TGETNUM :
Prototype : int tgetnum(const char *id);
Utilisation : La fonction tgetnum est utilisée pour récupérer une valeur numérique associée à une capacité spécifique
		      d'un terminal dans la base de données termcap. Par exemple, elle permet d'obtenir le nombre de colonnes
			  (co), de lignes (li), ou  la vitesse de transmission (pb). Elle est utile pour adapter l'affichage en fonction
			  des capacités du terminal.
Header : #include <termcap.h>
Parametre : id : Une chaîne de 2 caractères représentant le code de la capacité à récupérer (exemple :
			"co" pour le nombre de colonnes, "li" pour le nombre de lignes).
Retour : Retourne la valeur numérique de la capacité si elle est trouvée.
		 Retourne -1 si la capacité n'existe pas dans l'entrée du terminal.	




TGETSTR :
Prototype : char *tgetstr(const char *id, char **area);
Utilisation : La fonction tgetstr est utilisée pour récupérer une chaîne de caractères représentant une capacité spécifique
			  d'un terminal dans la base de données termcap. Ces chaînes sont généralement des séquences d'échappement
			  utilisées pour contrôler l'affichage du terminal (par exemple, déplacer le curseur, effacer l'écran,\
			  changer de couleur, etc.).
Header : #include <termcap.h>
Parametre : id : Une chaîne de 2 caractères représentant le code de la capacité à récupérer
			(exemple : "cl" pour effacer l'écran, "cm" pour positionner le curseur, "so" pour le mode standout).
			area : Un pointeur vers un tampon où la chaîne sera stockée. Ce tampon doit être géré par le programme appelant.
Retour : Retourne un pointeur vers la chaîne de caractères correspondant à la capacité demandée.
		 Retourne NULL si la capacité n'existe pas.




TGOTO : 
Prototype : char *tgoto(const char *cm, int dest_col, int dest_row);
Utilisation : La fonction tgoto est utilisée pour générer une séquence de contrôle permettant de positionner le curseur à une
			  position spécifique dans le terminal. Elle est souvent utilisée avec tgetstr("cm", ...) pour récupérer la
			  capacité "cm" (cursor motion) et ensuite l’appliquer aux coordonnées souhaitées.
Header : #include <termcap.h>
Parametre : cm : La chaîne de contrôle obtenue via tgetstr("cm", ...), qui contient le modèle de séquence d'échappement pour le déplacement du curseur.
			dest_col : La colonne de destination du curseur (coordonnée X).
			dest_row : La ligne de destination du curseur (coordonnée Y).
Retour : Retourne une chaîne de caractères contenant la séquence d'échappement formatée pour déplacer le curseur.
		 Retourne NULL si la capacité de positionnement du curseur n’est pas disponible.




TPUTS :
Prototype : int tputs(const char *str, int affcnt, int (*putc)(int));
Utilisation : La fonction tputs est utilisée pour envoyer une séquence d’échappement récupérée avec tgetstr,
			  tgoto ou tgetent vers le terminal. Elle permet d’appliquer correctement les capacités du terminal en tenant compte
			  des éventuels délais nécessaires.
Header : #include <termcap.h>
Parametre : str : La chaîne de contrôle du terminal à afficher (obtenue avec tgetstr, tgoto, etc.).
			affcnt : Le nombre de lignes affectées (souvent mis à 1, sauf pour les commandes qui influencent plusieurs lignes comme scroll).
			putc : Une fonction qui affiche un caractère (putchar, write, etc.).
Retour : Retourne 0 en cas de succès.
		 Peut retourner une autre valeur en cas d’erreur (mais ce comportement dépend de l’implémentation).
